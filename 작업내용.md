## 📸 19. 구글 드라이브 이미지 통합 저장 시스템 및 UI/UX 혁신 (2026-04-15)
Supabase Storage의 용량 제한을 극복하고 효율적인 사진 관리를 위해 구글 드라이브를 메인 이미지 저장소로 전면 전환했습니다.

### 구글 드라이브 기반 이미지 워크플로우
*   **Edge Function 통합 업로드**: `upload-daily-log-image` 함수를 통해 이미지를 구글 드라이브의 **'4. 첨부이미지모음'** 폴더에 자동 업로드합니다.
*   **지능형 파일명 자동 생성**: `거래처명_작성자_YYMMDD_HHMMSS.jpg` 형식으로 파일명을 자동 지정하여 관리 편의성을 극대화했습니다.
*   **공개 권한 자동화**: 업로드 즉시 파일 권한을 `anyoneWithLink` 및 `viewer`로 변경하여 별도의 권한 설정 없이 웹 사이트에서 즉시 조회가 가능하도록 구현했습니다.

### 성능 및 안정성 극대화
*   **일괄(Batch) 병렬 업로드**: 여러 장의 이미지를 동시에 서버로 전송하고 에지 함수 내부에서도 병렬로 업로드하도록 로직을 전면 개편하여 업로드 속도를 기존 대비 3배 이상 향상시켰습니다.
*   **브라우저 리사이징 및 압축**: Canvas API를 활용하여 전송 전 이미지를 **최대 가로 1200px**로 리사이징하고 화질을 **0.8** 수준으로 압축하여 모바일 업로드 속도를 비약적으로 향상시켰습니다.
*   **안정적인 미리보기 시스템**: 구글 드라이브의 보안 정책을 고려하여 `thumbnail?id=...&sz=w1000` 공식 엔드포인트를 사용함으로써 미리보기 이미지가 깨지는 현상을 완벽히 해결했습니다.

## 💎 20. 브랜드 정체성 강화 및 시스템 통합 디자인 (2026-04-15)
시스템의 전문성을 높이고 사용자 경험(UX)의 일관성을 확보하기 위해 브랜드 로고 및 페이지 명칭 체계를 전면 재정립했습니다.

### 로고 디자인 혁신 (COMTOOIN ITSM)
*   **신규 로고 개편**: 시스템 명칭을 **"COMTOOIN ITSM (IT Service Management)"**으로 변경하여 전문 IT 솔루션의 정체성을 강화했습니다.
*   **포인트 디자인 적용**: 로고 왼쪽에 차분한 **민트 에메랄드(#4db6ac)** 색상의 수직 포인트 바를 추가하여 시각적 엣지를 주었습니다.
*   **미니멀 화이트 테마**: 배경색인 Blue Grey와 조화를 이루도록 순백색과 투명도(Opacity) 그라데이션을 적용하여 고급스러운 ITSM 대시보드 느낌을 구현했습니다.
*   **반응형 로고**: 화면 크기에 따라 상세 설명을 자동으로 숨기거나 표시하는 지능형 타이포그래피 로직을 적용했습니다.

### 페이지 명칭 및 메뉴 구조 최적화
*   **직관적인 명칭 통일**: 모든 메뉴명과 페이지 메인 타이틀, 브라우저 탭 제목을 간결한 명사형(업무 기록, 대시보드, 리포트, 스케줄, 자료실, 거래처)으로 완벽하게 일치시켜 사용자 혼란을 방지했습니다.
*   **작업 흐름 기반 메뉴 재배치**: 실제 현장 업무 및 관리 동선을 고려하여 메뉴 순서를 **[업무 기록] → [대시보드] → [리포트] → [스케줄] → [자료실] → [거래처]** 순으로 최적화했습니다.
*   **디테일 UX 완성**: 
    *   자료실 빈 폴더 진입 시 불필요한 디버그용 ID를 제거했습니다.
    *   로딩 멘트를 **"자료실 목록을 불러오는 중입니다..."**로 변경하여 심리적 대기 시간을 완화했습니다.

## 📱 21. 대시보드 및 일정 상세 UI/UX 모바일 최적화 (2026-04-16)
현장 업무의 핵심인 모바일 환경에서의 조작 편의성과 정보 가독성을 극대화하기 위해 상세 정보창의 UI/UX를 전면 개편했습니다.

### 모바일 전체 화면 상세 시스템
*   **지능형 FullScreen 전환**: 대시보드 업무 상세 및 일정 상세 팝업을 모바일 화면(`sm` 이하)에서 자동으로 **전체 화면 모드**로 전환하여 작은 화면에서도 모든 정보를 시원하게 볼 수 있도록 구현했습니다.
*   **상단 핵심 정보 시각화**: 
    *   기존 한 줄로 뭉쳐 있던 거래처와 요청자 정보를 **두 줄 레이아웃**으로 분리했습니다.
    *   각 항목별 전용 아이콘과 `Caption + Body1 Bold` 조합의 폰트 시스템을 적용하여 가독성을 비약적으로 향상시켰습니다.

### 하단 버튼 레이아웃 표준화 (Design Consistency)
*   **통일된 버튼 배치 규칙**: 모든 상세 팝업의 하단 버튼을 **[삭제] (좌측 끝)** / **[닫기, 수정, 저장] (우측 그룹)** 순서로 완벽하게 통일하여 오클릭을 방지하고 작업 흐름의 일관성을 확보했습니다.
*   **모바일 최적화 버튼 디자인**: 
    *   전체 화면에 걸맞은 **기본 폰트 크기(`0.875rem`)**와 **충분한 터치 여백(`py: 1`)**을 확보했습니다.
    *   버튼 모서리 곡률을 **`borderRadius: 2`**로 높여 현대적이고 부드러운 느낌을 강조했습니다.
    *   모바일 환경에서 불필요한 아이콘을 숨겨 텍스트가 잘 보이고 버튼 배치가 밀리지 않도록 최적화했습니다.

## ✉️ 22. 다중 담당자 스케줄 이메일 발송 시스템 최적화 (2026-04-21)
스케줄 등록 시 1명 이상의 담당자에게 구글 캘린더 초대 메일이 정상적으로 발송되지 않던 문제를 해결하고 시스템의 안정성을 강화했습니다.

### 다중 담당자 초대 로직 혁신
*   **지능형 이메일 전처리**: 담당자가 여러 명일 때 발생할 수 있는 빈 이메일 주소나 중복된 이메일을 자동으로 필터링하여 구글 캘린더 API 호출 오류를 완벽히 차단했습니다.
*   **구글 API 규격 최적화 (All Day Event)**: '하루종일' 일정 등록 시 구글 API의 종료일(Exclusive) 처리 방식에 맞춰 시작일의 다음 날로 종료일을 자동 계산하도록 로직을 수정하여 일정 등록 실패 문제를 해결했습니다.
*   **안전한 업데이트 메커니즘**: 일정 수정(`PATCH`) 시 구글 이벤트 ID가 없는 예외 상황을 감지하여 자동으로 생성(`POST`)으로 전환함으로써 데이터 유실을 방지했습니다.

### 인프라 및 배포 환경 정형화
*   **CORS 보안 정책 완화**: 브라우저의 사전 요청(Preflight)과 에러 상황에서도 CORS 헤더가 확실히 포함되도록 응답 로직을 개선하여 서비스 안정성을 높였습니다.
*   **Supabase Edge Function 표준화**: 최신 `Deno.serve` 및 `std/http/server.ts` 라이브러리를 활용한 표준 코드로 전면 개편하여 환경 변화에 따른 오작동 가능성을 제거했습니다.
